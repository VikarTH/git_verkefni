Git spurningar og svör

1. Hvað gera eftirfarandi Linux skipanir?
	cd     skiptir úr möppuni sem maður er í í möppuna fyrir aftan
	cd ~   sendir notanda í heimamöppuna
	ls     sýnir allar möppur og skrár í möppuni
	pwd    sýnir leiðina að möppuni sem verið er í
	mkdir  býr til möppu

2. Hvað gera eftirfarandi git skipanir og hvers vegna eru þær gagnlegar?
	git clone     Býr til repository í nýrri möppu
	git log		  Sýnir commit sögu
	git status	  sýnir breytingar sem hafa verið gerðar frá seinasta push
	git diff	  sýnir breytingar á milli commits eða working tree
	git checkout  skiptir branch eða eyðir breytingum frá og með working tree skrám

3. Hver er munurinn á eftirfarandi git skipunum:
	a) git diff 					til að sjá breytingar sem hafa verið gerðar
	b) git diff --staged 			sýnir breytingar sem koma á næsta commit
	c) git diff commit1 commit2		ber saman breytingar frá commit1 og commit2

4. 	Hvað er útgáfustýring (e.version control)? 
	að velja útgáfu af einhverjum skrám
	
5.	Hverjir eru helstu kostir við að nota GIT?
	Að geta unnið með öðrum í sömu möppu og skráð allar breytingar frá notendum
	
6.	Hversu oft telur þú að eigi að gera commit í verkefni, rökstuddu?
	Eftir að maður hefur gert merkilega breytingu og veit að allt virki eða eftir 30 mín
	
7.	Hvað er "Working directory"?" "Staging area"?" og "Repository" í GIT?
	Working directory er mappan sem maður er í
	Staging area er næstu breytingar sem eiga eftir að vera commitaðar
	repository er endastaður fyrir allar breytingar sem maður vill gera
	
8.	Hvenær er sniðugt að nota greinar (branches)? 
	Þegar maður vill fara nýja leið með verkefni
